C++扫盲

C++
Publish Date:   2020-08-02
  1. 虚基类特点：基类被虚继承，无论在继承链中派生多少次，只有一个基类
  2. 虚继承
  3. 虚函数、虚析构函数指针在虚函数表中
  4. 在继承中class继承struct？或者struct继承class？合法吗？C++早期版本，不报错，但觉得不建议
  5. template < class Type > 与 template < struct Type > 编译器报错吗？ C++3.0都不报错，不推荐
  6. public、protected、private数据内存分布？ 按声明顺序分布，变量顺序编译器决定
  7. 将类中的很多数据传入C函数做法？ 数据封装为结构体，与类组合 子类继承结构体可行吗？ 新版编译器对继承布局做了调整，不建议
  8. 各种继承都支持多态吗？
  9. 访问类成员分类？ 类内部访问、子类访问、外部内访问
  10. 三种属性区别？ public三种都可访问 proteated外部不允许访问 private只允许内部访问
  11. C++编程范式？ 程序、模板、面向对象
  12. 多态实现原理？ 虚函数表
  13. 多态发生条件？ 继承 虚函数重写 父类调用子类资源
  14. 多态发生地点？ 发生了虚函数重写，继承，父类指使用子类资源
  15. 父类指针指向子类时，调用重写函数调用的是谁得函数？ 编译器识别父类指针，确定指针指向得空间大小为父类空间大小，指向了子类得父类资源，即子类得部分内存，所以调用的是父类得函数，而不是子类资源
  16. 父类指针static_cast/dynamic_cast转换为子类指针,调用重写函数，调用的是谁的？利用了什么机制？这两种转换方式有什么区别？ 编译器根据指针类型认为转换后得指针是子类指针，使用子类全部内存，调子类重写得函数。利用了virtual机制。static_cast编译时就转换完成，dynamic——cast执行时转换，资源消耗较静态转换大。
  17. virtual机制使用时是否发生了多态？ 没有，多态得实现使用的是虚函数表，利用了vptr指针，儿虚机制使用的是编译器根据指针类型决定内存的原理
  18. 各种变量得指针大小是多少？ 指针得大小与系统有关，32为系统4byte 64为系统8byte，任何变量都是大小固定的
  19. 类指针调用函数、类调用函数区别？ C++早期，指针调用使用虚函数表地址调用 类调用直接调用
  20. 表现一个类需要多大内存？ 非静态data成员总和 + 不够字节的data成员剩余空间 + 为virtual机制实现负担空间
  21. 多态中编译器如何知道自己需要解析的指针指向类的空间？ 根据指针类型
  22. 不适用virtual机制，子类指针使用父类初始化，可直接访问父类资源吗？ 不能，只能通过虚机制访问
  23. OB OO理解？ OB object base 不使用virtual机制 OO：使用virtual机制
  24. 使用. ->访问成员资源消耗差异？ 访问data成员由虚基类继承差异大
  25. 需要关注类型，直接继承，不关注类型，使用多态
  26. 静态成员函数访问非静态成员？ 不直接支持
  27. 静态成员函数与成员函数调用效率？为什么？ 相同 调用时编译器将静态的转换为非静态的，减小运行时间空间消耗， 非静态的成员函数会被转换为非成员函数
  28. 函数重载、虚函数重写如何编译？ 词法语法分析，根据特征为memberfunction改独一无二的名字
  29. 虚函数表基本信息？ 类型信息 虚函数指针
  30. 编译器编译时如何构建虚函数表，虚函数指针个数相同吗，多重继承呢？ 给每个虚函数留一个索引，表内装地址 个数相同 多重继承的类有多个表，对应父类，虚继承来的类只有一个表
  31. 虚函数编译时确定函数，虚函数指针能否实现多态，为什么？ 能 不同编译器实现机制不同，一种机制是将指针转换为结构体指针，结构体同时保存成员函数指针和虚函数指针，编译时判断类型
  32. 默认构造函数在什么时候被编译器合成？ 编译器认为在构造函数被需要的时候才会被合成
  33. 如果编译器没有构造函数，则合成构造函数，如果有则扩张该函数，放入默认构造函数需要的代码
  34. 编译器合成的默认构造函数只负责调用自身组合的类的默认构造函数
  35. 父类和成的默认构造函数会被子类默认构造函数调用
  36. 浅拷贝 深拷贝
  37. 使用函数内部的对象做返回值，返回的对象会不会被销毁？返回会不会调用拷贝构造函数？ 编译器会将函数重写，将内部返回的对象作为该函数引用参数传入，所以不会被析构。像这样的改写就不会调用拷贝构造，因此拷贝构造调用编译器而定，优化的越厉害调用可能性越小
  38. 拷贝传参会让编译器对函数做深奥的转化，效率明显没有引用高
  39. 构造函数使用快速初始化语法 A（int x）:i(x){y = i} i = x和 y = i谁先执行？ i=x先执行
  40. 空类有大小吗？ 有通常一个char大小，1 byte 为什么？ 为多个类对象定义独一无二的地址 虚继承的类的大小？非虚继承继承的呢？ 根据机器，编译器而定，为了vitual某些特殊支持
  41. 空类继承大小与编译器有关，非空类基层呢？ 无关
  42. 类的内存对齐？ 为每个类执行内存对齐，特别注意继承链上的也是为每个类执行内存对齐
  43. 虚函数多重继承，内部vptr表？ 早期方案 1.子类多表合一，通过vptr+offset存取 2子类内存布局特点，this+offset实现
  44. 指向成员函数的指针，不是用virtual 多重继承时效率与直接存取差不多
  45. ————构造析构拷贝语义学——————-
  46. 纯虚析构函数？是否能被调用？ 能，纯虚析构函数要有其实现体，不然在子类析构时无法调用父类析构函数，因此 需要在纯虚析构函数类外加上析构函数体 Base::~Base(){};
  47. 析构函数是否能被显式调用？ 能，不建议
  48. virtual会降低执行效率，不使用多态不建议使用
  49. 使用extren “C” 原因？ C++支持函数重载，C++编译器更改某些函数的名字更与C编译器不同
